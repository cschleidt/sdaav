Motivation

The objective of the project is to identify a location for a new bar in New York. We want to find a "sweet spot" where there is an active party scene but in a "nice" area. The reader of this document should get a basic understanding of New Yorks areas zip codes and understand why we choose to open a bar in the chosen location.

The data

The "Party Bar" dataset indicates where the party scene is, while a combination of other attributes will guide us to the "nice" neighbourhoods. Data for the project for based on NYC OpenData and .... PUT REFERRENCE HERE ... NYC OpenData provides different statistics,

  1. Rodent Complaints
  2. Graffiti Complaints
  3. Air quality Complaints
  4. Urinating in Public Complaints
  5. Property Market Value
  6. Number of trees
  7. Condition of trees (share of trees that are in good condition)

Exploring the data

The visualization below gives an explanatory overview of data used for the analysis. It provides the user with a basic understanding of New Yorks areas zip codes and the data/parameters that is the basis for our model. The diagram outlines the datasets used in the analysis as mentioned above:

  • Rodent Complaints
  • Graffiti Complaints
  • Air quality Complaints
  • Urinating in Public Complaints
  • Property Market Value
  • Number of trees
  • Condition of trees (share of trees that are in good condition)

To make data comperable they are all scaled (0-1), where 0 illustrates a minimum value and 1 a maximun value within the zipcodes.

Select the buttons to the right to display the different catetogies.

Cluster map

Using the values found and shown in the data exploration section we defined a combined value for the complaints, the market pricing and the life quality of the trees. We call this value "Niceness" which is an indicator from 0-1 which shows how pleasant a zip code is compared to the others.

Additionally we grouped the zip codes together into 20 clusters, which have similar trademarks of complaints.

We used these to show the clusters by color, niceness and the relative party potential on the bars on the left.

Using this illustration you can look at the individual zip codes to see how it stands in comparison to the rest

Scatterplot

Looking at the previous illustration we have chosen the cluster we deemed to be best in regards to both niceness and music.

These have been plotted on scatterplot, and the change over time is shown by moving the slider or automatically by cliking the button

Each circle represents a zip code, and the size represent the combined value of both niceness and music
